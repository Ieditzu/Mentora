# Mentora protocol v1 golden vectors

`packets.json` is the canonical byte-level contract shared by the Java
backend, Unity game, Android client, and Kotlin/iOS shared client.

The file is generated by `generate_vectors.py`, a deliberately independent
Python implementation. It does not import production code from any Mentora
client.

## Decrypted packet payload

All backend protocol numbers are signed and big-endian.

```text
[int32 packet ID]
[packet-specific body]
```

Primitive fields are encoded as follows:

- `int32`: four signed big-endian bytes
- `int64`: eight signed big-endian bytes
- `bool`: one byte, `0x00` or `0x01`
- `string`: `[int32 UTF-8 byte count][UTF-8 bytes]`
- repeated values: `[int32 count][encoded item 0]...[encoded item n]`
- binary data: `[int32 byte count][raw bytes]`

String lengths count UTF-8 bytes, not Unicode characters. Several vectors
therefore contain Romanian diacritics, emoji, and newlines.

## Encrypted WebSocket envelope

```text
[int32 encrypted-seed byte count]
[16-byte seed IV][AES-256-CBC encrypted int64 seed]
[16-byte payload IV][AES-256-CBC encrypted packet payload]
```

- AES padding is PKCS#7. Java's `PKCS5Padding` name is equivalent for AES.
- The seed encryption key is `SHA-256(baseKeyUtf8)`.
- The payload encryption key is
  `SHA-256(ASCII decimal representation of the decrypted seed)`.
- The plaintext seed is a signed big-endian `int64`.

The committed encrypted vectors use one fixed seed and fixed IVs solely for
repeatable tests. Production encoders must continue generating fresh seeds
and random IVs.

## Test expectations

Each implementation should test both halves independently:

1. Decode `encryptedEnvelopeBase64` with production code and assert the
   semantic fields.
2. Encode the semantic packet with production code, decrypt it with a
   test-only reference decoder, and compare the resulting bytes with
   `plainPayloadBase64`.

Comparing a production encoder only with its own production decoder is not a
cross-language compatibility test.

`unityDecode` and `unityEncode` describe current Unity support. Packets
81–92 are canonical TOTP and parent-session contracts but are not registered
by the game.

Packet ID 1 intentionally has both a legacy v1 vector and a v2 vector. The v2
body appends `[int32 protocolVersion][string deviceId]`; a missing tail means
protocol version 1 and an empty device ID.

## Regeneration

The generator requires Python 3 and `cryptography`.

```bash
python3 test-fixtures/protocol/v1/generate_vectors.py --write
python3 test-fixtures/protocol/v1/generate_vectors.py --check
```

Review semantic fields and byte changes together. A protocol change should
increment `protocolVersion` or add a new fixture directory rather than
silently rewriting an incompatible v1 field order.
